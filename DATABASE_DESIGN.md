# FlatFund - Firebase Database Design

## 🗂️ Firestore Collections Structure

### 1. **`users`** Collection
Stores user/member profiles and authentication data.

```typescript
users/{userId}
{
  uid: string;                    // Firebase Auth UID
  email: string;                  // User email
  name: string;                   // Full name
  phone?: string;                 // Phone number (optional)
  avatar?: string;                // Avatar URL or initials
  role: "admin" | "member";       // User role
  isActive: boolean;              // Active status
  
  // Financial tracking
  monthlyTarget: number;          // Monthly contribution target (e.g., 1500)
  
  // Metadata
  createdAt: Timestamp;           // Account creation date
  joinedAt: string;               // Join date (YYYY-MM format)
  lastLogin?: Timestamp;          // Last login time
  
  // Settings
  twoFactorEnabled: boolean;      // 2FA status
  notificationPreferences: {
    email: boolean;
    deposits: boolean;
    expenses: boolean;
    bills: boolean;
  };
}
```

**Example Document:**
```json
{
  "uid": "abc123xyz",
  "email": "john@example.com",
  "name": "John Doe",
  "phone": "+91 9876543210",
  "avatar": "JD",
  "role": "member",
  "isActive": true,
  "monthlyTarget": 1500,
  "createdAt": "2024-01-15T10:30:00Z",
  "joinedAt": "2024-01",
  "twoFactorEnabled": false,
  "notificationPreferences": {
    "email": true,
    "deposits": true,
    "expenses": true,
    "bills": true
  }
}
```

### 2. **`deposits`** Collection
Tracks all money deposited into the central fund.

```typescript
deposits/{depositId}
{
  id: string;                     // Auto-generated ID
  userId: string;                 // Reference to users/{userId}
  userName: string;               // Denormalized for quick display
  userAvatar?: string;            // Denormalized avatar
  
  amount: number;                 // Deposit amount
  date: string;                   // Date (YYYY-MM-DD)
  month: string;                  // Month (YYYY-MM) for filtering
  
  reference?: string;             // Transaction reference
  note?: string;                  // Description
  status: "pending" | "confirmed"; // Payment status
  
  // Auto-deposit tracking
  isAutoGenerated: boolean;       // True if created from self-expense
  linkedExpenseId?: string;       // If auto-generated, link to expense
  
  createdAt: Timestamp;           // Record creation time
  updatedAt: Timestamp;           // Last update time
}
```

**Example Document:**
```json
{
  "id": "dep_1234567890",
  "userId": "abc123xyz",
  "userName": "John Doe",
  "userAvatar": "JD",
  "amount": 1500,
  "date": "2024-01-15",
  "month": "2024-01",
  "reference": "UPI/123456",
  "note": "Monthly contribution",
  "status": "confirmed",
  "isAutoGenerated": false,
  "createdAt": "2024-01-15T10:30:00Z",
  "updatedAt": "2024-01-15T10:30:00Z"
}
```

### 3. **`expenses`** Collection
Tracks all expenses (both fund and self).

```typescript
expenses/{expenseId}
{
  id: string;                     // Auto-generated ID
  userId: string;                 // Who made the expense
  userName: string;               // Denormalized
  userAvatar?: string;            // Denormalized
  
  amount: number;                 // Expense amount
  date: string;                   // Date (YYYY-MM-DD)
  month: string;                  // Month (YYYY-MM) for filtering
  
  type: "fund" | "self";          // Expense type
  category: string;               // Category name (e.g., "Groceries")
  categoryIcon?: string;          // Icon name
  
  note: string;                   // Description
  receipt?: boolean;              // Has receipt attached
  receiptUrl?: string;            // Receipt image URL (if uploaded)
  
  status: "confirmed" | "reimbursement_requested" | "reimbursement_approved" | "reimbursement_rejected";
  
  // Auto-generation tracking
  isAutoGenerated: boolean;       // True if auto-created fund expense
  linkedExpenseId?: string;       // Link to original self-expense
  
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### 4. **`bills`** Collection
Tracks shared recurring bills.

```typescript
bills/{billId}
{
  id: string;                     // Auto-generated ID
  title: string;                  // Bill name (e.g., "Electricity")
  type: string;                   // Bill type
  icon?: string;                  // Icon name
  
  amount: number;                 // Total bill amount
  perMemberShare: number;         // Amount per member
  
  dueDate: string;                // Due date (YYYY-MM-DD)
  billingMonth: string;           // Billing period (YYYY-MM)
  
  paidBy?: string;                // User ID who paid
  paidByName?: string;            // Denormalized name
  status: "pending" | "paid" | "overdue";
  
  invoiceNumber?: string;         // Invoice/bill number
  
  // Track individual member payments
  memberPayments: {
    [userId: string]: "paid" | "pending" | "partial";
  };
  
  createdAt: Timestamp;
  updatedAt: Timestamp;
}
```

### 5. **`settings`** Collection
Global app settings (single document).

```typescript
settings/appSettings
{
  general: {
    currency: string;             // "INR"
    timezone: string;             // "Asia/Kolkata"
    dateFormat: string;           // "DD/MM/YYYY"
    defaultTarget: number;        // Default monthly target
    lowFundThreshold: number;     // Alert threshold
  };
  
  categories: Array<{
    id: number;
    value: string;
    label: string;
    icon: string;
    color: string;
    isDefault: boolean;
  }>;
  
  policies: {
    allowPendingSpends: boolean;
    requireReceipts: boolean;
    autoApproveReimbursements: boolean;
    lockWindow: number;           // Days after which records lock
  };
  
  updatedAt: Timestamp;
  updatedBy: string;              // User ID who last updated
}
```

## 🔐 Updated Firestore Security Rules

```javascript
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    
    // Helper functions
    function isSignedIn() {
      return request.auth != null;
    }
    
    function isOwner(userId) {
      return request.auth.uid == userId;
    }
    
    function isAdmin() {
      return isSignedIn() && 
             get(/databases/$(database)/documents/users/$(request.auth.uid)).data.role == 'admin';
    }
    
    // Users collection
    match /users/{userId} {
      allow read: if isSignedIn();
      allow create: if isSignedIn() && isOwner(userId);
      allow update: if isOwner(userId) || isAdmin();
      allow delete: if isAdmin();
    }
    
    // Deposits collection
    match /deposits/{depositId} {
      allow read: if isSignedIn();
      allow create: if isSignedIn() && request.resource.data.userId == request.auth.uid;
      allow update: if isOwner(resource.data.userId) || isAdmin();
      allow delete: if isAdmin();
    }
    
    // Expenses collection
    match /expenses/{expenseId} {
      allow read: if isSignedIn();
      allow create: if isSignedIn() && request.resource.data.userId == request.auth.uid;
      allow update: if isOwner(resource.data.userId) || isAdmin();
      allow delete: if isAdmin();
    }
    
    // Bills collection
    match /bills/{billId} {
      allow read: if isSignedIn();
      allow create, update: if isSignedIn();
      allow delete: if isAdmin();
    }
    
    // Settings collection
    match /settings/{document=**} {
      allow read: if isSignedIn();
      allow write: if isAdmin();
    }
    
    // Monthly stats (read-only for members)
    match /monthlyStats/{month} {
      allow read: if isSignedIn();
      allow write: if isAdmin();
    }
    
    // Reimbursements
    match /reimbursements/{reimbursementId} {
      allow read: if isSignedIn();
      allow create: if isSignedIn() && request.resource.data.userId == request.auth.uid;
      allow update: if isAdmin();
      allow delete: if isAdmin();
    }
  }
}
```

## 📋 What You Need to Do

### **Step 1: Set Up Firebase Console**

1. **Go to Firebase Console**: https://console.firebase.google.com/
2. **Select your project**: `flatfund-af16a`

### **Step 2: Enable Firestore Database**

1. In the left sidebar, click **"Firestore Database"**
2. Click **"Create database"**
3. Choose **"Start in production mode"** (we'll add rules next)
4. Select your region: **asia-south1** (Mumbai) or closest to you
5. Click **"Enable"**

### **Step 3: Add Security Rules**

1. In Firestore Database, go to the **"Rules"** tab
2. **Copy the entire security rules** from the section above
3. **Paste** into the rules editor
4. Click **"Publish"**

### **Step 4: Enable Authentication**

1. In the left sidebar, click **"Authentication"**
2. Click **"Get started"**
3. Go to **"Sign-in method"** tab
4. Enable these providers:
   - ✅ **Email/Password** (click, toggle enable, save)
   - ✅ **Google** (optional, for social login)

### **Step 5: Create Initial Settings Document**

1. In Firestore Database, go to **"Data"** tab
2. Click **"Start collection"**
3. Collection ID: `settings`
4. Document ID: `appSettings`
5. Add these fields:

```
Field: general (map)
  ├─ currency: "INR"
  ├─ timezone: "Asia/Kolkata"
  ├─ dateFormat: "DD/MM/YYYY"
  ├─ defaultTarget: 1500
  └─ lowFundThreshold: 1000

Field: categories (array)
  [Leave empty for now, will be populated by app]

Field: policies (map)
  ├─ allowPendingSpends: false
  ├─ requireReceipts: false
  ├─ autoApproveReimbursements: false
  └─ lockWindow: 3

Field: updatedAt (timestamp)
  [Click "Set to current time"]

Field: updatedBy (string)
  "system"
```

### **Step 6: Set Up Indexes (Optional but Recommended)**

Firestore will prompt you to create indexes when needed, but you can create them in advance:

1. Go to **"Indexes"** tab
2. Click **"Add index"**

**Recommended Indexes:**

**For `deposits`:**
- Collection: `deposits`
- Fields: `month` (Ascending), `createdAt` (Descending)

**For `expenses`:**
- Collection: `expenses`
- Fields: `month` (Ascending), `type` (Ascending), `createdAt` (Descending)

**For `bills`:**
- Collection: `bills`
- Fields: `billingMonth` (Ascending), `status` (Ascending)

### **Step 7: Configure Firebase in Your App**

Your Firebase config is already set up in `client/lib/firebase.ts`. You're good to go!

### **Step 8: Test the Setup**

1. Try creating a test user via Authentication
2. Manually add a test document to `users` collection
3. Verify security rules work by trying to read/write

## 📊 Database Size Estimation

**For a typical flat with 5 members:**
- Users: 5 documents
- Deposits: ~150/month (30 per member)
- Expenses: ~200/month
- Bills: ~10/month
- Settings: 1 document

**Monthly data:** ~365 documents  
**Yearly data:** ~4,380 documents

**Firebase Free Tier:**
- 1 GB storage (plenty for years)
- 50K reads/day
- 20K writes/day

You'll stay well within free tier limits! 🎉
